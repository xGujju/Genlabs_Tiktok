# Format Experiments

Durable notes for GenLabs AI Learning format tests beyond static carousel. Optimize for views, follower growth, saves, shares, completion/swipe-through, and repeatable production speed.

## 2026-04-27T16:03Z — Next experiment: Narrated mistake→fix carousel

### Recommendation
Run a **narrated mistake→fix carousel** in the next 24–48h: keep the current Thai hand-drawn/sketchnote carousel asset workflow, but add a 45–60 second Thai voiceover/screen-record-style narration and a mistake-to-fix story arc.

### Why this format is the best next step
- It is the lowest-risk bridge from static carousel to motion: no full video shoot, no face cam, no heavy editing, and reusable carousel assets.
- It targets TikTok signals beyond simple swipes: stronger first 2–3 seconds, completion, saves/shares, spoken keyword/search context, and comment prompts.
- It matches GenLabs constraints: Thai-first, beginner-friendly, draft-only/manual-review, visual-first slides, broad AI usefulness, and 10 quality drafts/day pressure.
- It can be logged cleanly as one format experiment while preserving normal Airtable entities: source signal → Content Pipeline → Assets → Experiments → Performance.

### Research anchors used
- YouTube Shorts official creator page: Shorts can be made with phone-native tools, captions, simple vertical clips, and no fancy equipment; useful for lightweight narration/motion tests.
- Buffer short-form video explainer: short-form works best when vertical, focused, authentic to niche, and educational as well as entertaining.
- Hootsuite TikTok views guide: hook in the first few seconds, create curiosity, promise value, and use captions to clarify why viewers should keep watching.
- Sprout Social TikTok algorithm guide: TikTok uses completion, saves/shares, comments, captions/hashtags/sounds, and search-value signals; photo mode/carousels benefit from complete swipe-through.
- Social Media Examiner short-form video guide: keep one main point and one CTA; optimize hook, pattern interruption, captions, and cover/first frame.

### Hypothesis
If GenLabs turns a useful AI tip into a narrated mistake→fix carousel, then views and follows should improve versus static carousel because the format combines:
1. a stronger pain-point hook,
2. a story loop viewers want resolved,
3. audio/caption search signals,
4. save-worthy step-by-step utility,
5. low production cost that can scale.

### Experiment design
- Test window: next 24–48h.
- Minimum posts: 3 concepts using the same structure, one per topic cluster.
- Control/comparison: nearest static carousel posts on similar AI-usefulness topics.
- Primary metrics: views, follows per 1,000 views, average watch time/completion if video analytics are available, carousel completion/swipe-through if photo-mode analytics are available.
- Secondary metrics: saves, shares, comments, profile visits.
- Winning threshold: outperform recent static carousel median on views OR follows/1k views, while not taking more than ~1.5× static carousel production time.

### Exact template
Format: TikTok photo carousel or short vertical video assembled from carousel slides + Thai voiceover.
Length target: 45–60 seconds, 6 slides, 1 idea only.
Visual style: GenLabs hand-drawn Thai sketchnote; 80% graphic / 20% text; one short headline + 1–3 labels per slide.

1. **Slide 1 — Mistake hook**
   - Text: “พลาดแบบนี้ AI เลยตอบมั่ว” / topic-specific version.
   - Visual: red X, confused robot, messy prompt/document.
   - Voiceover: name the mistake in one sentence.
2. **Slide 2 — Why it hurts**
   - Text: “เสียเวลา + ได้คำตอบกว้าง”
   - Visual: before result is vague/incorrect.
   - Voiceover: explain consequence for everyday user/business.
3. **Slide 3 — The fix**
   - Text: “แก้ด้วยสูตร 3 ช่อง”
   - Visual: simple box formula: Context → Goal → Output.
   - Voiceover: introduce fix clearly.
4. **Slide 4 — Example before/after**
   - Text: “ก่อน / หลัง”
   - Visual: bad prompt vs improved prompt, minimal text.
   - Voiceover: read only the key change, not all text.
5. **Slide 5 — Copyable mini-template**
   - Text: “ก๊อปสูตรนี้ไปใช้”
   - Visual: template card.
   - Voiceover: tell viewers to save.
6. **Slide 6 — Follow/comment CTA**
   - Text: “อยากได้สูตรงานอะไรต่อ?”
   - Visual: comment bubble + GenLabs robot.
   - Voiceover: “คอมเมนต์งานที่อยากให้ทำให้ดู เดี๋ยวทำตอนต่อไป”

Caption pattern: 1-line Thai hook + 3 bullet steps + CTA to save/comment + max 5 hashtags.
Comment seed: ask a specific work-context question, not generic “คิดเห็นยังไง”.

### 3 Thai concept candidates for this format
1. **“พิมพ์ถาม ChatGPT แบบนี้ เลยได้คำตอบกว้างมาก”**
   - Mistake: ถามสั้นเกินไป เช่น “ช่วยเขียนโพสต์ขายของให้หน่อย”
   - Fix: ให้บริบทสินค้า + ลูกค้า + โทน + รูปแบบผลลัพธ์
   - Follow hook: “ติดตามไว้ เดี๋ยวทำสูตร prompt สำหรับแม่ค้าออนไลน์ต่อ”
2. **“ก่อนส่งเอกสารให้ AI ลบ 3 อย่างนี้ก่อน”**
   - Mistake: โยนเอกสารจริงที่มีเบอร์โทร เลขบัตร ที่อยู่ เข้า AI
   - Fix: ปิด/แทนข้อมูลส่วนตัวก่อน แล้วค่อยให้ AI สรุป
   - Follow hook: “คอมเมนต์ว่าใช้ AI กับเอกสารแบบไหน เดี๋ยวทำเช็กลิสต์ให้”
3. **“ทำรูปสินค้าแล้วดูปลอม เพราะสั่ง AI ผิดจุด”**
   - Mistake: ขอให้ AI “ทำรูปให้สวย” แบบไม่มีสไตล์/แสง/ฉาก/ข้อห้าม
   - Fix: ใช้สูตร Subject + Background + Lighting + Do not change
   - Follow hook: “เซฟไว้ใช้กับรูปสินค้า และคอมเมนต์หมวดสินค้าที่อยากให้ทำตัวอย่าง”

### Production checklist
1. Pick source signal from AI Signals/current trend scout, preferably workflow/prompt/tool utility.
2. Write one mistake→fix script before generating images.
3. Generate/reuse 6 carousel slides in GenLabs visual style.
4. Record Thai voiceover or TTS; add burned-in captions if assembling as video.
5. Upload as draft only; no auto-publish.
6. Log format = `narrated_mistake_fix_carousel` in Experiments/Performance.

### Risks and controls
- Risk: voiceover adds delay. Control: cap script at 120–150 Thai words and use TTS if needed.
- Risk: too much text ruins readability. Control: slide text stays headline-level; details move to voice/caption.
- Risk: topic too niche for mass Thai audience. Control: choose everyday/business pain: prompts, privacy, images, documents, selling, studying.
- Risk: video assembly blocks 10 drafts/day. Control: test only 3 posts first; do not replace the static carousel pipeline until metrics justify it.

### Durable rule
For GenLabs, the first step beyond static carousel should not be high-production video. The repeatable bridge is **static carousel assets + narration + a mistake→fix or before/after story arc**, because it tests retention and follow conversion while preserving the existing carousel factory.
